var, let, and const are used to declare variables in JavaScript. The main difference is how they behave with scope and reassignment. var is function-scoped and can cause unexpected bugs because it ignores block scope and is hoisted. let is block-scoped, meaning it only works inside { }, and its value can be changed. const is also block-scoped but cannot be reassigned after declaration. In modern JavaScript, developers usually prefer using let and const for cleaner and safer code. 

The spread operator (...) is used to expand or copy elements from arrays or objects. It helps make code shorter and easier to read. For example, you can copy an array, merge multiple arrays, or pass array values into a function easily. Instead of writing loops to copy data, the spread operator allows quick duplication or combination of values in one line. It is very useful when working with API data or updating arrays in modern JavaScript applications.

In real projects, map() is used for transforming data, filter() for selecting specific data, and forEach() mainly for running code on each element. 

An arrow function is a shorter way to write functions in JavaScript. It uses the => syntax instead of the traditional function keyword. Arrow functions make code cleaner and easier to read, especially for small functions. They are commonly used in modern JavaScript and frameworks like React. One important difference is that arrow functions do not have their own this value, which can change how they behave inside objects or event handlers. 

Template literals are a modern way to write strings in JavaScript using backticks ( `` ). They allow dynamic values to be inserted into strings using${}` syntax. This makes it easier to combine variables and text without complicated string concatenation. Template literals also support multi-line text, which improves readability when building UI messages or HTML content. They are widely used in modern JavaScript for creating dynamic text.
